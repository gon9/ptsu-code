# E2E評価: 実用性テスト (Day 1-2 完了後)

## 目的

Phase 1 で実装した基本的な Agentic Loop（Tool Calling + マルチプロバイダー対応）を、
実際のユースケースで操作し、実用性・UX・不足機能を評価する。

---

## 評価対象機能

| 機能 | 実装状況 | 評価項目 |
|------|---------|---------|
| OpenAI/Anthropic プロバイダー切替 | ✅ | 切替がスムーズか、エラーハンドリングは適切か |
| ファイル読み書きツール | ✅ | 実際のコードベースで使えるか |
| コマンド実行ツール | ✅ | タイムアウト処理、エラー表示は適切か |
| マルチターン Tool Calling | ✅ | 複数ツールを連続使用できるか |
| CLI UX (Rich UI) | ✅ | 見やすさ、操作性 |

---

## ユースケースシナリオ

### シナリオ 1: コードベース理解（Read-only）

**目的**: エージェントがプロジェクト構造を理解し、質問に答えられるか検証

**操作手順**:
```bash
uv run ptsu chat --provider openai
```

**対話例**:
```
You > このプロジェクトの構成を教えて

期待動作:
1. list_directory または read_file で README.md を読む
2. src/ ディレクトリを探索
3. プロジェクト構成を要約して返答

評価ポイント:
- ツール選択は適切か
- 複数ファイルを読んで統合できるか
- 応答が分かりやすいか
```

```
You > agent/runtime.py の主要な機能は？

期待動作:
1. read_file で runtime.py を読む
2. クラス・メソッドを解析
3. 主要機能を説明

評価ポイント:
- ファイルパスの推測は正確か
- コード理解の深さ
```

### シナリオ 2: 簡単なコード修正（Write操作）

**目的**: ファイル書き込みとツール連携を検証

**操作手順**:
```bash
uv run ptsu chat --provider anthropic
```

**対話例**:
```
You > tests/agent/tools/test_base.py にカバレッジを上げるテストを追加して

期待動作:
1. read_file でテストファイルを読む
2. read_file で対象の base.py を読む
3. 不足しているテストケースを特定
4. write_file で新しいテストを追加

評価ポイント:
- 既存コードを壊さないか
- テストの品質
- ユーザーに変更内容を説明するか
```

### シナリオ 3: コマンド実行とフィードバックループ

**目的**: コマンド実行 → 結果分析 → 再実行のループを検証

**操作手順**:
```bash
uv run ptsu chat
```

**対話例**:
```
You > テストを実行して、失敗があれば修正して

期待動作:
1. execute_command("uv run pytest")
2. 失敗があれば、エラーメッセージを分析
3. read_file で該当ファイルを読む
4. write_file で修正
5. execute_command で再テスト

評価ポイント:
- エラー分析の精度
- 修正の適切さ
- ループが無限に続かないか（max_turns で止まるか）
```

### シナリオ 4: プロバイダー切替

**目的**: OpenAI と Anthropic の動作差を検証

**操作手順**:
```bash
# OpenAI で実行
uv run ptsu chat --provider openai
You > src/ptsu_code/config.py に新しい設定項目を追加して

# Anthropic で同じタスク
uv run ptsu chat --provider anthropic
You > src/ptsu_code/config.py に新しい設定項目を追加して
```

**評価ポイント**:
- 両プロバイダーで同等の結果が得られるか
- Tool Calling の安定性
- レスポンス速度の違い

### シナリオ 5: エラーハンドリング

**目的**: 異常系での振る舞いを検証

**操作手順**:
```bash
uv run ptsu chat
```

**対話例**:
```
You > 存在しないファイル /tmp/nonexistent.txt を読んで

期待動作:
1. read_file を実行
2. エラーが返る
3. エージェントがエラーを理解し、ユーザーに報告

評価ポイント:
- エラーメッセージが分かりやすいか
- エージェントがパニックしないか
```

```
You > 無限ループするコマンドを実行して

期待動作:
1. execute_command でタイムアウト
2. タイムアウトエラーが返る
3. エージェントが適切に報告

評価ポイント:
- タイムアウトが正しく動作するか
- プロセスが残らないか
```
